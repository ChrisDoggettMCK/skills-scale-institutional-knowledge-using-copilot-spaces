# OctoAcme Project Management Docs

## Overview

This directory is the single entry point for OctoAcme's project management documentation. These docs describe how OctoAcme plans, executes, and continuously improves cross-functional projects that deliver product features, services, and integrations.

## Principles

OctoAcme's project management is guided by six core principles:

- **Customer-first value delivery**: prioritize work that delivers clear value to customers.
- **Incremental and testable releases**: deliver small, reviewable increments rather than large batches.
- **Clear ownership and accountability**: every project has a named Project Manager and Product Lead.
- **Data-informed decision making**: measure outcomes and iterate based on evidence.
- **Transparent risk management and communication**: surface and track risks early; keep stakeholders informed.
- **Learning and continuous improvement**: capture retrospective insights and act on them.

## Lifecycle

Projects move through five stages:

1. **Initiation** — validate the idea, align stakeholders, and produce a lightweight one-pager/charter.
2. **Planning** — define scope, milestones, resource needs, dependencies, and a risk register.
3. **Execution & Tracking** — build, test, review, and iterate; run standups and weekly syncs; maintain the backlog and Definition of Done.
4. **Release & Deployment** — deploy, verify, publish release notes, and communicate to stakeholders.
5. **Retrospective / Improvement** — document learnings, update processes, and carry action items forward.

## Key Artifacts and Cadence

**Artifacts**: project one-pager/charter, roadmap and release plan, sprint/iteration backlog, Definition of Done and acceptance criteria, risk register, release notes, retrospective notes and action items.

**Team cadence**: twice-weekly standups (or as agreed), weekly PM + PdM sync, sprint/iteration demos, monthly stakeholder updates, and ad-hoc escalations as needed.
